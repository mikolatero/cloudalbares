\section{Introducción} 

\subsection{¿Que es un sistema de virtualización?}

\begin{frame}\frametitle{¿Que es un sistema de virtualización?} 

\begin{itemize}

\item La creación -a través de software- de una versión virtual de algún recurso tecnológico, como puede ser una plataforma de hardware, un sistema operativo, un dispositivo de almacenamiento u otros recursos de red \pause
\bigskip

\item Una máquina virtual simula una plataforma de hardware autónoma incluyendo un sistema operativo completo que se ejecuta como si estuviera instalado. Típicamente varias máquinas virtuales operan en una misma maquina fisica. Para que el sistema operativo ?guest? funcione correctamente, la maquina fisica debe ser lo suficientemente potente para realizar esta simulacion. \pause
\bigskip

\item Existen diferentes proveedores de software para virtualizar: \medskip
\begin{itemize}
 \item VMware  \medskip
 \item Virtualbox   \medskip
 \item Parallels \medskip
 \item Xen \medskip
 \item KVM  \medskip
\end{itemize}

\end{itemize}

\end{frame}


\subsection{¿Que es el cloud?}

\begin{frame}\frametitle{¿Que es el cloud?} 

\begin{itemize}

\item Cloud Computing permite ofrecer de manera conveniente y a demanda, recursos y servicios altamente disponibles tanto desde la red privada como la red pública.    \pause
\bigskip

\item Permite a los usuarios acceder a un catálogo estándar de servicios, respondiendo a las necesidades del negocio de forma flexible y permitiendo adaptarse a las demandas de los usuarios en el tiempo.
\pause
\bigskip

\item 
Su principal característica es el uso de recursos compartidos y la independencia de la infraestructura, lo cual permite al usuario disponer siempre de los mismos servicios independientemente donde este se encuentre ubicado.
\pause
\bigskip

\item Las características de Cloud Computing son:

\begin{itemize}
 \item Servicio a demanda \medskip
 \item Pool de recursos independiente de la ubicación  \medskip
 \item Elasticidad y flexibilidad   \medskip
 \item Servicio medible 
\end{itemize}

\end{itemize}

\end{frame}



\subsection{Tipos de Cloud}

\begin{frame}\frametitle{Tipos de Cloud}



\begin{block}{Privado}
Infraestructura on-demand implementada y administrada exclusivamente por la organización, que controla que servicios ofrecer .  Una Cloud Privada es una la mejor opción para las compañías que necesitan una alta protección de datos y asegurar el acceso continuo a los servicios sin la dependencia del acceso a Internet. 
\end{block}
\pause
\bigskip
\begin{block}{Publico}
Infraestructura on-demand implementada de manera que puede ser accedida de manera pública y desde internet.     
	\begin{itemize}
		\item Generalmente ofrecida por empresas de comunicación y datacenter.
	    \item Servicios accedidos de manera estándar desde internet.
	\end{itemize}
\end{block}


\end{frame}

\begin{frame}\frametitle{Tipos de Cloud}

\begin{block}{Hibrido}
Se combinan los modelos de nubes públicas y privadas, donde el cliente es propietario de una parte de la infraestructura, mientras comparte otra.  Este modelo de Cloud es el que en teoría sería el más atractivo para las organizaciones y el que más se desarrollara con el tiempo.  Una Cloud hibrida trabaja bajo el concepto de federación y portabilidad.
\end{block}


\end{frame}

\section{Diferentes soluciones de virtualizacion} 

\subsection{OpenNebula}
\begin{frame}\frametitle{OpenNebula} 
\begin{itemize}
\item  OpenNebula ofrece la solución abierta, más adaptable y con mayor funcionalidad para construir centros de datos virtualizados y nubes privadas con Xen, KVM y VMware en entornos empresariales
\pause \bigskip
\item  OpenNebula es un proyecto muy activo con una amplia base de usuarios
\pause \bigskip
\item  Incluyen los principales centros de investigación como CERN, Fermilab y ESA; centros de supercomputación como SARA y NCHC; operadores de telecomunicaciones como RIM, China Mobile y Telefónica O2, e integradores como Logica, Engineering y KPMG. 
\pause \bigskip
\item C12G Labs, la compañía que gestiona el proyecto de código open-source, ofrece una distribución comercial de OpenNebula para las organizaciones que buscan una plataforma de nube certificada y con soporte a largo plazo. OpenNebula y C12G son marcas comerciales de C12G Labs

\end{itemize}

\end{frame}

\subsection{OpenStack}
\begin{frame}\frametitle{OpenStack} 
\begin{itemize}
\item  En los inicios OpenStack era un cloud propio de RackSpace creando dos servicios 
\begin{itemize}
\item Cloud Servers
\item Cloud Files
\end{itemize}
\pause \bigskip
\item  Este software cambia a licencia libre en Abril 2010
\pause \bigskip
\item En este momento Interviene la NASA y Comienzan a utilizar Eucalyptus, pero lo descartan por no ser completamente libre (es open Core).
\pause \bigskip
\item La NASA crea el software para IaaS Nebula
\pause \bigskip
\item Nebula cambia a licencia libre en Mayo 2010
\pause \bigskip
\item Nasa y Rackspace Inician OpenStack en Junio de 2010, sus dos componentes principales son 
\begin{itemize}
\item OpenStack Compute (nova), deriva de Nebula
\item OpenStack Object Store (swift), deriva de cloud Files
\end{itemize}

\end{itemize}

\end{frame}

\subsection{RedHat Enterprise Virtualization}
\begin{frame}\frametitle{RedHat Enterprise Virtualization} 
\begin{itemize}
\item  El esquema de funcionamiento es igual que en VMware y XenServer
\pause \bigskip
\item  Necesitando de un centro/s de almacenamiento compartido (NFS, ISCSI ó FC) para el alojamiento de nuestras VMs
\pause \bigskip
\item Necesita  2 o más NODOS para poder hacer migraciones
\pause \bigskip
\item Para la creación de las VMs, necesitaremos obligatoriamente un Almacenamiento NFS para que albergue nuestras ISOs
\pause \bigskip
\item Red Hat Enterprise Virtualization es también un componente esencial de Red Hat Cloud Infrastructure
\pause \bigskip
\item La version 3.2 de RHEV ofrece las siguientes mejoras
\pause \bigskip
\begin{itemize}
\item Compatibilidad completa con Storage Live Migration
\item Soporte para los últimos procesadores estándar de Intel y AMD
\item Mejoras en gestión de almacenamiento, gestión de redes y energía, fencing, mejoras en la consola Spice, registro, monitorización y muchas mas...
\end{itemize}

\end{itemize}

\end{frame}

\subsection{Microsoft Windows Server 2012}
\begin{frame}\frametitle{Microsoft Windows Server 2012} 
\begin{itemize}
\item Funcionalidades Principales 
\pause \bigskip
\begin{itemize}
\item La virtualización de servidores: Hyper-V proporciona herramientas para la creación y administración de un entorno de servidor virtualizado. Hyper-V Replica puede replicar una máquina virtual a partir de un lugar a otro, sin almacenamiento compartido, proporcionando una mayor recuperación de desastres y alta disponibilidad.
\pause \bigskip
\item Gestión del servidor: El Core Server proporciona un entorno mínimo para ejecutar funciones específicas de servidor. Estos roles son entonces más fáciles de mantener y gestionar y menos expuesto a los ataques. Estas funciones incluyen Active Directory y servicios relacionados, DHCP, DNS, servicios de archivos, Hyper-V, Servicios de impresión, Streaming Media Services y Web Server (IIS).
\pause \bigskip
\item Servicios integrados: Windows Server 2012 proporciona una plataforma unificada para la publicación web que integra IIS, ASP.NET y Windows Communication Foundation y también es compatible con SharePoint Services.
\end{itemize}
\end{itemize}
\end{frame}

\begin{frame}\frametitle{Microsoft Windows Server 2012} 
\begin{itemize}
\item Funcionalidades Principales 
\pause \bigskip
\begin{itemize}
\item  Failover Clustering: Los administradores pueden minimizar las interrupciones de los servicios y aplicaciones mediante la creación de un entorno de servidor redundante con particiones de hardware dinámica.
\pause \bigskip
\item Servidor de aplicaciones: esta función de servidor proporciona un entorno integrado para implementar y ejecutar aplicaciones empresariales personalizadas creadas con Microsoft NET Framework 3.5.
\pause \bigskip
\item DirectAccess: Esta característica túnel seguro permite el acceso a la red desde cualquier ordenador o dispositivo. Es más rápido que las típicas conexiones VPN y ofrece fuera de la zona de acceso a los recursos compartidos de archivos, equipos en las instalaciones y otros recursos.

\end{itemize}
\end{itemize}

\end{frame}
